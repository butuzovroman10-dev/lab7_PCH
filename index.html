🔧 Структура узла

struct Node {
    T data;
    Node* prev;
    Node* next;
    
    Node(const T& value) 
        : data(value), prev(nullptr), next(nullptr) {}
};

📊 Поля контейнера

Node* head;      // указатель на первый элемент
Node* tail;      // указатель на последний элемент
size_t count;   // количество элементов
Compare comp;    // компаратор (std::less<T> по умолчанию)
✅ Ключевая особенность: При добавлении элементов через push_back() они автоматически вставляются в отсортированном порядке. Контейнер всегда остаётся отсортированным без необходимости вызывать sort().

⚙️ Алгоритм вставки insertSorted()

void insertSorted(Node* newNode) {
    // 1. Если список пуст
    if (!head) {
        head = tail = newNode;
        return;
    }
    
    // 2. Вставка в начало (новый элемент меньше первого)
    if (comp(newNode->data, head->data)) {
        newNode->next = head;
        head->prev = newNode;
        head = newNode;
        return;
    }
    
    // 3. Вставка в конец (новый элемент больше последнего)
    if (!comp(newNode->data, tail->data)) {
        tail->next = newNode;
        newNode->prev = tail;
        tail = newNode;
        return;
    }
    
    // 4. Вставка в середину (поиск позиции)
    Node* current = head;
    while (current && comp(current->data, newNode->data)) {
        current = current->next;
    }
    
    if (current) {
        newNode->prev = current->prev;
        newNode->next = current;
        if (current->prev) {
            current->prev->next = newNode;
        }
        current->prev = newNode;
    }
}
⏱️ Сложность вставки: O(n) для поиска позиции + O(1) для вставки = O(n). По сравнению с исходной реализацией (массив) — вставка не требует сдвига элементов, что экономит время на копировании.

➡️ Итератор (Bidirectional Iterator)

class Iterator {
private:
    Node* ptr;
    
public:
    // Категория итератора - двунаправленный
    using iterator_category = std::bidirectional_iterator_tag;
    using value_type = T;
    using difference_type = std::ptrdiff_t;
    using pointer = T*;
    using reference = T&;
    
    // Операторы
    reference operator*() const;
    Iterator& operator++();    // переход к следующему элементу
    Iterator& operator--();    // переход к предыдущему элементу
    bool operator==(const Iterator&) const;
};
⚠️ Отличие от исходной реализации: Итератор теперь имеет категорию bidirectional_iterator_tag (вместо random_access_iterator_tag). Это означает, что операции it + 5 и it[3] больше не поддерживаются, так как список не предоставляет произвольный доступ к элементам.

✅ Преимущества и ❌ Недостатки

✅ Преимущества

  • Автоматическая сортировка – элементы всегда упорядочены
  • Эффективная вставка/удаление – O(1) после нахождения позиции
  • Отсутствие переаллокаций – не нужно изменять размер массива
  • Стабильность итераторов – не инвалидируются при операциях
  • Динамический размер – без ограничений по ёмкости

❌ Недостатки

  • Медленный доступ по индексу – O(n) вместо O(1)
  • Дополнительная память – на хранение prev/next указателей
  • Невозможен бинарный поиск – нет произвольного доступа
  • Плохая локальность данных – узлы разбросаны по памяти
  • Поиск всегда O(n) – даже для отсортированных данных

Условие:

В IT-компании работают сотрудники с разными ID. Необходимо:

  1. Хранить сотрудников в отсортированном по ID порядке (автоматически)
  2. Найти сотрудника по ID с использованием std::find_if
  3. Удалить сотрудника по ID
  4. Вывести всех сотрудников с эффективностью выше 85%

🔍 Использование std::find_if

// Поиск сотрудника по ID
auto it = std::find_if(container.begin(), container.end(), 
    [id](Employee* emp) { 
        return emp->getId() == id; 
    });

if (it != container.end()) {
    std::cout << "Found: ";
    (*it)->displayInfo();
}

// Удаление найденного сотрудника
container.remove(*it);
delete *it;
💡 Важно: Несмотря на то, что контейнер всегда отсортирован, std::find_if работает за O(n), так как итератор не поддерживает произвольный доступ. Для бинарного поиска потребовалась бы структура с произвольным доступом (массив) или специализированный алгоритм.
